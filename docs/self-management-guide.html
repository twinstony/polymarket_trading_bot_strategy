Polymarket 脚本自主管理说明

以后你自己管理脚本时，主要只需要改市场、方向、买入价、投入金额、卖出价和交易开关。钱包配置通常不要动。

最核心的一句话： 每次交易前确认市场没结束，算好份数，填好卖出价，把 TRADING_ENABLED=true 打开；比赛结束或暂停排查时改回 false。

3. 每次交易主要改这些

字段 作用 填写方式
TRADING_ENABLED交易总开关true 允许交易；false 禁止自动买卖
TOKEN_ID具体方向的 outcome token必须是你要买的那一边
MARKET_LABEL日志显示名写清楚比赛、市场、方向
SHARE_AMOUNT买入份数投入金额 / 买入价
ENTRY_PRICE买入限价例如 0.35
EXIT_PRICE卖出目标价例如 0.65
CONDITIONAL_ENTRY是否等待更好价格false 启动就挂；true 等盘口达到条件

4. 份数计算

公式：

SHARE_AMOUNT = 投入 USDC / 买入价格
投入买入价份数
50 USDC0.25200
50 USDC0.35142.86
30 USDC0.15200
40 USDC0.20200

5. 单层交易

适合一笔买入、一笔卖出目标。

TRADING_ENABLED=true
TOKEN_ID=目标方向token
MARKET_LABEL=比赛 - 市场 - 方向
SHARE_AMOUNT=份数
ENTRY_PRICE=买入价
EXIT_PRICE=卖出价
CONDITIONAL_ENTRY=false

运行：

cd /Users/ad/Documents/polydata/polymarket_trading_bot_strategy
source .venv/bin/activate
python main.py

停止：在终端按 Ctrl + C。

6. 分层交易

适合同一个方向多档买入，例如 0.35 买 50 和 0.25 买 50，同一个 0.65 卖出目标。

cd /Users/ad/Documents/polydata/polymarket_trading_bot_strategy
source .venv/bin/activate
python tools/layered_order_runner.py \
  --market-slug 市场slug \
  --token-id 目标方向token \
  --label "日志名称" \
  --exit-price 0.65 \
  --layer 0.35:50 \
  --layer 0.25:50 \
  --state-file .runtime/本次交易.json \
  --poll-interval 20

--layer 0.35:50 表示 0.35 买入 50 USDC。

分层监控器会记录旧仓位基线，只给本次新增成交份数补卖单。

7. 比赛结束后

比赛结束或不再适合入场时，先关交易总开关。 
TRADING_ENABLED=false
  • 停止正在跑的脚本或 screen。
  • 去 Polymarket 的 Portfolio -> Open 检查是否有不需要的买单。
  • 已经持有的仓位可以继续保留卖单。
  • 比赛结束后不能再补买单。

8. 每次运行前检查

  • 市场没有结束。
  • 整场、Game 1、Game 2 没有选错。
  • 方向没有选错，比如 DK 和 Gen.G。
  • TOKEN_ID 是正确 outcome token。
  • SHARE_AMOUNT 已按投入金额换算。
  • EXIT_PRICE 已填写。
  • 运行后去 Polymarket Open 页面确认订单。

9. 最高优先级风控

  • 不能只按 token 判断是否买过。
  • 下单前必须查挂单、成交、持仓。
  • 网络/API 不确定时暂停，不补单。
  • 买入成交后必须同步挂卖单。
  • 同 token 分层加仓时，新增份数必须单独补卖单。
  • 比赛结束后禁止延迟买入。
