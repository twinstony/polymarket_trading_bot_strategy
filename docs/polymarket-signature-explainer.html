Polymarket 挂单与钱包签名字段说明

本页记录这次脚本挂单、Rabby 钱包签名字段，以及以后排查时最容易混淆的几个地址。

这次挂单是成功的。

Portfolio 的 Open 列表里显示：Buy Gen.G 25¢，数量 0 / 160，总额 $40.00，过期时间 Until cancelled。

0 / 160 表示当前还没有成交，订单总量是 160 shares。只要它在 Open 列表里，就是一个正在等待成交的挂单。

三个地址不要混

Rabby 外部钱包地址

0xYourRabbySignerAddress

由当前 PRIVATE_KEY 推导出来，负责签名。

Polymarket API / Funder 地址

0xYourPolymarketFunderAddress

脚本里的 FUNDER。Rabby 签名弹窗里的 maker、signer、verifyingContract 都指向它。

充值地址

只用于向 Polymarket 充值，不填到脚本的 FUNDER。

Rabby 签名字段说明

字段 意思 这次截图里的含义
Operation 钱包要执行的签名类型。 TypedDataSign，表示签的是 EIP-712 typed data，不是直接转账。
contents.salt 订单随机数/唯一编号，用来防止同一订单签名被重复使用。 每次新订单通常都会不同。
maker 订单资金/仓位归属地址。 应等于 Polymarket API / Funder 地址。
signer 订单声明的签名主体。 SIGNATURE_TYPE=3 时应等于 Polymarket API / Funder 地址。
tokenId 具体 outcome token 的 ID。一个市场通常有多个 outcome token。 决定买的是哪一边，例如 Gen.G、KC、YES 或 NO。
makerAmount 买单时通常是愿意支付的 USDC 金额，按 6 位小数放大。 76000000 约等于 76 USDC。具体数值取决于网页那次订单的价格和数量。
takerAmount 买单时通常是想买到的 shares 数量，按 6 位小数放大。 100000000 约等于 100 shares。
side 买卖方向。 0 表示 BUY，1 表示 SELL。
signatureType Polymarket 钱包签名模式。 3 表示 POLY_1271 / deposit wallet。
timestamp 订单签名时间戳，通常是毫秒。 用于校验订单签名生成时间。
metadata 订单附加元数据。 全 0 表示没有额外元数据。
builder 构建者/推荐方字段。 全 0 表示没有 builder 标识。

Domain 字段说明

字段 意思
name: DepositWallet 说明这次签名是 deposit wallet 体系下的签名。
version: 1 签名域版本。
chainId: 137 Polygon 主网。
verifyingContract 负责验证签名的合约地址。对于 deposit wallet，通常应等于 Polymarket API / Funder 地址。
salt 签名域的 salt。截图里是全 0，属于正常情况。

以后怎么看是否成功

  • 终端里看到 Order placed、success: True、status: live，说明脚本提交订单成功。
  • 网页 Portfolio 的 Open 里出现订单，说明订单正在挂着等待成交。
  • Filled 0 / 160 表示还没有成交；如果变成 160 / 160，表示全部成交。
  • 要停止脚本，在终端按 Ctrl + C。
安全提醒：钱包弹窗里如果 maker、signer、verifyingContract 不是你当前 Polymarket API / Funder 地址，先不要签。私钥不要保存进截图、文档或聊天记录。
